CSS Engine — Beta Status Report

This document records the current state of the CSS subsystem for the v0.1.0-beta release. It is the single source of truth for what works, what is partial, and what is missing.

Build Verified: webcore target builds cleanly (100% success on 2026-06-12). All CSS .cpp files compile without errors.

Implementation Status by Feature

Cascade & Matching COMPLETE

CSSCascade, SelectorMatcher, specificity, origin sorting, and indexed lookup are fully implemented and tested in unit suites.

Style Resolution COMPLETE

StyleResolver + applyProperty covers all standard properties, length parsing, color parsing, and inheritance. Computed style cache is live.

Parser PARTIAL

Full rule/declaration parsing, @import/@media/@keyframes/@font-face/@supports supported. Missing: container queries, nesting, @layer, @scope (ES2024+).

Layout Integration PARTIAL

Block, flex, and grid layout engines exist and consume computed styles. Missing: subgrid, masonry, container queries, writing-mode vertical.

Animations & Transitions MISSING

No Web Animations API, no CSS transition/animation engine, no requestAnimationFrame-driven interpolation yet.

Variables & @supports COMPLETE

CSS custom properties and @supports condition evaluation are wired in CSSEngine.

Critical Beta Blocker

The JS-DOM Bridge is only a header.
source/webCore/scripting/js_dom_bridge.hpp declares JSDOMBridge, wrapElement, exposeDocumentObject, etc., but there is no corresponding .cpp implementation and no concrete DOM classes (DOMElement, DOMDocument) wired to ZepraScript.

Result: Even though CSS and layout are production-grade, any page that relies on JavaScript (Next.js, React, Vue, Angular, SPAs) renders as a blank or broken shell because the DOM never materializes.

Recommendations for Beta

  1. Document the gap publicly — users must know that only static HTML/CSS sites will render correctly in v0.1.0-beta.
  2. Implement a minimal DOM stub (even read-only) so that at least document.createElement and basic element creation work for simple JS.
  3. Wire the bridge before the next milestone so that React hydration can begin.
  4. Keep CSS frozen — it is the one subsystem that is genuinely production-ready; do not touch it until the JS-DOM bridge lands.
